# Code Review

Use this skill when the user asks for a review or when validating a risky change.

Lead with findings. Prioritize correctness bugs, behavioral regressions, security
or data-loss risk, and missing tests. Cite precise files and lines when available.

Do not spend the review budget on style unless it creates real maintenance or
runtime risk. If no issues are found, say that clearly and name residual test
coverage gaps.
